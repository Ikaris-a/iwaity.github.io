访问一个Web页面经历过程

当你在地址栏输入一个URL的时候,这时候的URL会带着信息去从服务器上面获取资源信息,最终把资源传给Web浏览器。从而展示出页面,这个过程就是从客户端到服务端,再从服务端到客户端的运作流程。Web就是建立在HTTP协议的通信

TCP/IP

TCP/IP协议族分层

当然,要了解HTTP,也要知道TCP/IP协议族。
其实HTTP是属于这个协议族的一个子集。

TCP/IP协议族可分为四层:

  • 应用层:
    向用户提供应用服务时通信的活动 eg:FTP-DNS
  • 传输层:
    给处于网络链接中的两台计算机提供数据传输
  • 网络层:
    处理网络流动的数据包
  • 数据链路层:
    处理链接网络的硬件部分 eg:网卡,光纤

DNS服务

DNS(Domain Name System)服务也是位于应用层的协议,主要负责域名到IP地址之间的解析服务.人类的记忆习惯善于运用图形化和其他记忆方式。所以相对来说数字的ip地址相对来说很难记得.
比如说让你记得谷歌的域名: google 和谷歌访问ip地址: 93.123.23.1 要好得多！DNS协议就通过域名查找ip,或者逆向从ip去查域名的服务

URI与URL

URL是Uniform Resource Locator的缩写,译为”统一资源定位符”.
URL 是 URI 的子集.任何东西,只要能够唯一地标识出来,都可以说这个标识是 URI.
URI分为URL和URN这两大类.

  • URI: Uniform Resource Identifier.统一资源标识符;
  • URL: Uniform Resource Locator.统一资源定位符;
  • URN: Uniform Resource Name.统一资源名称.
